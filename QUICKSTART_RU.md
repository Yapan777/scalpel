# Scalpel — Быстрый старт

Этот гайд проведёт тебя от нуля до первого запуска. Всё работает локально — никакого облака, никаких API-ключей.

---

## Что тебе понадобится

| Требование | Минимум | Рекомендуется |
|---|---|---|
| RAM | 16 ГБ | 32 ГБ |
| Python | 3.10+ | 3.11 |
| CPU | любой | 8+ ядер |
| GPU | не нужен | — |
| ОС | Windows / Linux / macOS | — |

---

## Шаг 1 — Скачай репозиторий

```bash
git clone https://github.com/ТВО_ИМЯЮЗЕРА/scalpel.git
cd scalpel
```

Или скачай ZIP через GitHub → **Code → Download ZIP**, распакуй и войди в папку.

---

## Шаг 2 — Установи Python-зависимости

```bash
pip install -e ".[full]"
```

Это установит всё сразу: numpy, scipy, scikit-learn, PySR и DSPy.

> **Если хочешь только базовые зависимости без PySR/DSPy:**
> ```bash
> pip install -e .
> ```

---

## Шаг 3 — Установи Ollama и скачай модели

Ollama — локальный сервер для языковых моделей. Scalpel использует 9 моделей разных архитектур.

### 3.1 Установи Ollama

Зайди на [ollama.ai](https://ollama.ai) и скачай установщик под свою ОС. После установки проверь:

```bash
ollama list
```

Должен появиться пустой список (или список уже скачанных моделей).

### 3.2 Скачай все модели

```bash
ollama pull qwen2.5:7b
ollama pull deepseek-r1:7b
ollama pull phi4:14b
ollama pull mistral:7b
ollama pull granite3.1-dense:8b
ollama pull llama3:8b
ollama pull gemma2:9b
ollama pull yi:9b
ollama pull internlm2:7b
```

> Каждая модель весит 4–8 ГБ. Суммарно ~50 ГБ. Загрузку можно оставить на ночь.
>
> Если RAM не хватает — достаточно первых 4 моделей (qwen, phi4, mistral, llama3). Остальные роли система заменит ими.

## Шаг 4 — Первый запуск

### Быстрый тест (рекомендуется начать здесь)

```bash
python run_real.py --subset 5 --noise gaussian --seed 42
```

Берёт первые 5 законов Фейнмана, добавляет 10% гауссов шум, запускает полный цикл поиска.

**Это займёт несколько часов** — каждый закон ~25 минут PySR + время LLM-ролей.

Если запуск прервался — просто запусти ту же команду снова. Система продолжит с места остановки.

### Все варианты запуска

```bash
# Все 122 закона Фейнмана (полный прогон)
python run_real.py

# Первые N законов
python run_real.py --subset 5

# Конкретный тип шума
python run_real.py --noise gaussian    # гауссов шум 10%
python run_real.py --noise outliers    # выбросы
python run_real.py --noise hetero      # гетероскедастичность
python run_real.py --noise missing     # пропущенные значения

# Без --noise: все 4 типа по кругу (один тип на один закон)
python run_real.py --subset 5

# Количество точек (система адаптирует параметры автоматически)
python run_real.py --subset 5 --n_samples 200

# Зафиксировать случайность для воспроизводимости
python run_real.py --subset 5 --seed 42
```

## Шаг 5 — Что ты увидишь в конце

По каждому закону система печатает результат:

```
──────────────────────────────────────────────────────────────
  I.12.1: [СКРЫТО] | 4 признаков | шум: gaussian
──────────────────────────────────────────────────────────────
  Данные: 150 train / 50 test (оба зашумлены)
  ...
  ✅ [ПРИНЯТО] I.12.1
     Нашла:    (f1 / f4) / (f1 + (f4 * (f3 + (3.6387281 - f2))))
     Истина:   [СКРЫТО → real_ground_truth_PRIVATE.json]
     R²=0.9874  Консенсус=ПРИНЯТА  t=1823.4s
     Шум: gaussian
```

В конце всего прогона — итог:

```
══════════════════════════════════════════════════════════════
  REAL SCIENCE MODE — ИТОГ
══════════════════════════════════════════════════════════════
  ✅ Принято:   3/5
  🟡 Кандидат: 0/5
  ❌ Отклонено:2/5
──────────────────────────────────────────────────────────────
  Шум: 10% (gaussian)
  Результаты → scalpel_vault/real_results
  Истина     → real_ground_truth_PRIVATE.json
══════════════════════════════════════════════════════════════
```

> `real_ground_truth_PRIVATE.json` — файл с правильными ответами для 5 законов Фейнмана из этого запуска. **Не влияет на поиск формул** — система его не читает во время работы. Он нужен только как журнал ответов: система сама дописывает в него новые законы после каждого запуска, как учитель заполняет тетрадь с ответами уже после того как ученик сдал работу.

Файлы с результатами:

```
scalpel_vault/
├── real_results/
│   └── real_results_2026-04-08.json  ← результат текущего запуска
├── FINAL_REPORT_LLM_v10.txt          ← итоговый отчёт
├── gold_formulas.json                 ← все принятые формулы
├── reasoning_log.jsonl                ← полный лог рассуждений каждой роли
└── CONSENSUS_REPORT.txt              ← последний консенсус Матрёшки
```

## Частые проблемы

**Ollama не отвечает**
```bash
ollama serve   # запусти в отдельном терминале
```

**Julia не установлена / PySR падает**

PySR установит Julia автоматически при первом запуске. Нужен интернет. Если оффлайн:
```bash
set JULIAPKG_OFFLINE=yes      # Windows
export JULIAPKG_OFFLINE=yes   # Linux / macOS
```

**Мало RAM — система падает на phi4:14b**

phi4 весит 14B параметров. Если RAM < 16 ГБ:
```bash
python -m scalpel --model mistral:7b
```

**Прервал запуск на середине**

Просто запусти ту же команду снова — система найдёт файл с промежуточными результатами и продолжит с места остановки.
